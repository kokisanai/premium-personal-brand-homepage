# Premium Personal Brand Homepage

Use this plugin when the user wants a premium, conversion-focused personal brand homepage for an author, speaker, consultant, founder, or expert operator.

## Behavior

Build the page as a real personal-brand website, not a generic SaaS landing page. Center the experience around the person's positioning, proof, offers, credibility assets, and conversion paths.

Prioritize:
- A strong first viewport with a human presence, concrete proof, and clear primary/secondary CTAs.
- Real assets over decorative placeholders: portraits, speaking photos, book covers, newsletter screenshots, client/media logos, proof screenshots, and video thumbnails.
- Editorial hierarchy, strong contrast, compact rhythm, and solid text surfaces when imagery is nearby.
- Conversion modules for booking, subscribing, buying books, watching talks, or contacting the person.

Avoid:
- Fake logos, guessed contact details, invented metrics, lorem ipsum, or generic feature-card filler.
- Replacing real portraits, book covers, or client proof with generated visuals.
- Adding 3D, particles, charts, maps, workflow libraries, or motion systems unless they solve a clear communication problem.
- Letting image layers, canvases, textures, or motion cover readable copy.

## Workflow

1. Read the user's brief, provided references, and any existing artifact files.
2. Extract stable facts: name, role, audience, credibility proof, offers, books/products, newsletter/community, testimonials, and conversion destinations.
3. Decide which sections need real assets and which should stay type/diagram-led. Do not force imagery into every section.
4. Download or embed only reliable public assets. If an asset cannot be verified, use a clear `needs manual asset` label rather than a fake substitute.
5. Design a responsive homepage with these default sections when relevant:
   - Hero
   - Trust/proof bar
   - Entry cards or offer shortcuts
   - Problem / market moment
   - Offers
   - Books / products / media
   - Newsletter or audience capture
   - Testimonials / proof
   - Final CTA
6. Add restrained interactivity only when it improves comprehension or conversion: mobile nav, cards/tabs, copy-to-clipboard, form validation, scroll reveal, or light contextual motion.
7. Verify before handoff:
   - Headings wrap intentionally.
   - Real assets remain intact and readable.
   - Logos are correct for the exact company.
   - Text is not covered by images or motion layers.
   - CTAs are separated clearly from surrounding controls.
   - Mobile layout is not just a squeezed desktop layout.
